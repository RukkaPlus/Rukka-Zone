    Syntax

    • Es un lenguaje de programacion multiparadigma
    • Es un lenguaje de programacion dinamico y de tipado debil
    • Los comentarios se manejan con #
    • Utiliza la indentacion como separador de bloques de codigo, y esta debe ser de 4 espacios por nivel
    • Soporta Unicode
    • Los nombres de las variables no pueden comenzar con numeros, ni pueden tener un keyword como nombre ni tener espacios entre sus nombres

    Instalacion

    Al ser Python un lenguaje de programacion interpretado, es necesario contar con un interprete para poder ejecutar codigo de Python en un sistema
    Podemos conseguir y adquirir su interprete desde la pagina oficial de Python, mediante gestores de versiones y distribuciones de entornos Python (por ejemplo: Anaconda) o mediante manejadores de paquetes (por ejemplo: APT (Debian GNU/Linux), YUM (Fedora GNU/Linux), PACMAN (Arch GNU/Linux)), entre otros

    Utilizacion

    Una vez instalado el interprete de Python, podremos ejecutar un script/programa de Python mediante el siguiente comando

    > py direccion\del\script.py
    $ python3 direccion/del/script.py

    Para mas informacion referente a la utilizacion de la aplicacion de linea de comandos de Python, dirijase a la documentacion oficial de utilizacion de la aplicacion

    pip

    Generalmente, junto a la instalacion de Python viene incluido un gestor de paquetes llamado pip
    pip permite importar/instalar, desarrollar y exportar/distribuir paquetes entre otras cosas, por lo que es una herramienta bastante util al momento de desarrollar paquetes de Python

    Utilizacion

    Una vez instalado Python deberiamos poder preparar y actualizar pip en el sistema con el siguiente comando

    Para actualizar/instalar pip ejecutamos el siguiente comando

    > py -m pip install --upgrade pip

    Para actualizar/instalar pip ejecutamos el siguiente comando

    $ python3 -m pip install --upgrade pip

    Si obtiene un error al ejecutar el comando, porfavor, consulte la documentacion oficial de Python

    Nota: Algunas herramientas que puede instalar pueden venir en formato de binarios. Programas, utilidades o aplicaciones que se ejecutan directamente desde la linea de comandos. Para ejecutarlos correctamente, asegurese de que la carpeta de binarios de Python se encuentra entre sus variables de entorno. En algunos sistemas Linux, puede agregar export PATH=$PATH:/home/coffee/.local/bin a su archivo ~/.bash_profile

    Virtual Environments

    Los Virtual Environments (en espaniol, Entornos Virtuales) son, como su nombre lo indica, entornos virtuales que uno puede preparar para desarrollar un Software en concreto.
    En estos Virtual Environments uno puede instalar o preparar paquetes aislados del sistema con el fin de prevenir problemas de compatibilidad o conflictos con versiones de dependencias de un Software

    Utilizacion

    Una vez instalado Python, por lo general, tendremos dos utilidades que nos permitiran manejar Virtual Environments: venv y virtualenv
    En este ejemplo, utilizaremos el tradicional virtualenv, que es bastante completo y recomendado

    Para crear una Virtual Environment ejecutamos el siguiente comando

    > py -m virtualenv VIRTUAL_ENVIRONMENT_NAME

    Seguido a ello, para activar o iniciar el Virtual Environment usualmente ejecutamos el siguiente comando

    > VIRTUAL_ENVIRONMENT_NAME\Scripts\activate.bat

    Para ver mas opciones relacionadas, puede ejecutar el siguiente comando

    > py -m virtualenv --help

    Para crear una Virtual Environment ejecutamos el siguiente comando

    $ python3 -m virtualenv VIRTUAL_ENVIRONMENT_NAME

    Seguido a ello, para activar o iniciar el Virtual Environment usualmente ejecutamos el siguiente comando

    $ source /VIRTUAL_ENVIRONMENT_NAME/bin/activate

    Para ver mas opciones relacionadas, puede ejecutar el siguiente comando

    $ python3 -m virtualenv --help

    Nota: Estas herramientas pueden ser ejecutadas mediante una consola (por ejemplo, cmd, PowerShell, Sakura, GNOME terminal, Konsole, etc)

    Funciones

    Para definirlo, debemos anteponer el keyword def al nombre de esta funcion. Seguido a ello, opcionalmente podemos definir los parametros que recibira (o definir que no requiera parametros)

    # Para definir una funcion
def funcion_0():
# Es importante indentar el codigo
# para separarlo como bloque de codigo
return 'Hello, world!'
# 'return' indica el valor que sera
# devuelto por la funcion

# Para definir una funcion con parametros
def funcion_1(parm_0, parm_1 = 2):
# Podemos requerir cuantos parametros
# deseemos. Y como en el caso de 'parm_1',
# indicar su valor por defecto en caso de 
# que este no sea ingresado
return f'''Hello, {parm_0}!
You have {parm_1} apples'''

# Para definir una funcion 'lambda'
funcion_lambda = lambda num_0, num_1: num_0 + num_1

# Para ejecutar funciones
return_0 = funcion_0()
return_1 = funcion_1('RukkaPlus', 24)
return_lambda = funcion_lambda(12, 52)
print(return_0)
print(return_1)
print(return_lambda)

    Bucles

    Los bucles son estructuras de control de flujo que repiten un bloque de codigo una cantidad determinada de veces
    En Python tenemos dos estructuras de bucle: For y While

    For

    El bucle For itera un dato (como una lista) y ejecuta un bloque de codigo (con el dato temporalmente almacenado) hasta terminar de recorrer todos los datos del iterable

    comidas = [
  'Leche', 'Pan', 'Queso',
  'Galletas', 'Cafe',
]
for comida in comidas:
  print(comida)

    El bucle While ejecuta un bloque de codigo en base a una condicion. Siempre que la condicion establecida sea cumplida, repetira el bloque de codigo indicado

    variable = 0
while variable > 6:
  print('Estoy dentro del bucle WHILE')
  variable += 1
print('He salido del bucle WHILE')

    Keywords utilizados en bucles

    • continue : Repite el bucle
    • break : Rompe el bucle y continua con el programa

    Manejo de excepciones

    El manejo de excepciones nos permite controlar el flujo y comportamiento del programa a partir de la causa una excepcion. Esto lo podemos manejar con las estructuras try, catch, except, finally u raise

    Keywords utilizados en el manejo de excepciones

    • try : Ejecuta un bloque de codigo propenso a errores
    • except : Captura un error producido dentro del try y ejecuta un bloque de codigo
    • finally : Ejecuta un bloque de codigo haya o no ocurrido un error
    • raise : Produce y arroja un error
    try: # Ejecuta un codigo propenso a errores...
  print('Este bloque producira un error')
  raise SyntaxError('Soy un error!!!') # Esto arroja un error

except: # Si ocurre un error...
  print('Ha ocurrido un error durante la ejecucion del programa!')

else: # Si no ocurre un error...
  print('Todo ha funcionado correctamente')

finally: # Ocurra o no ocurra un error...
  print('Programa ha finalizado')

    Manejo de mutiples errores

    Es posible manejar multiples errores en caso de que hayan sido ocurridos durante el programa

    from random import random

  valor = random()

    raise TypeError('Ocurrio un error de tipo [TypeError]')

  raise ValueError('Ocurrio un error de tipo [ValueError]')
except TypeError: # Si produce un error de tipo [TypeError]...
  print("Error: ocurrio un error de tipo!")

    Modulos

    Los Modulos son las Librerias/Bibliotecas de Python.
    Un Modulo, en conceptos simples, es un programa o script de Python creado que puede ser importado por otros programas o scripts con la intencion de reutilizar conceptos como funciones, variables, clases, entre otras.
    Para importar un modulo, podemos utilizar la keyword import nombre_del_modulo o from nombre_del_modulo import datos_de_importacion
    Tenemos los siguientes tipos de modulos

    • Modulos de Python : Estos vienen por defecto por Python y pueden ser importados en cualquier programa o script
    • Modulos propios : Son modulos que nosotros desarrollamos y que podemos importar dentro de nuestro programa siempre que lo tengamos disponible
    • Modulos terceros : Son modulos desarrollados por terceros y que podemos importar dentro de nuestro programa siempre que lo tengamos disponible

    Utilizacion de modulos de Python

    Al venir por defecto integrado en el entorno de Python, podemos importarlos en cualquier programa o script sin importar su ubicacion

    # | main.py |
# [ Importacion de un modulo ]
import math   # Importacion del modulo 'math'
math.sqrt(16) # Utilizacion de la funcion 'sqrt' del modulo 'math'

# [ Importacion de conceptos de un modulo ]
from math import sqrt # importacion directa de 'sqrt' del modulo 'math'
sqrt(16)              # Ya no es necesario anteponer 'math.'

    Utilizacion de modulos propios

    Tambien podemos importar modulos propios dentro de un programa o script de Python si lo tenemos disponible. Osea, si lo tenemos instalado con pip o indicamos su ubicacion exacta

    # | my_module.py |
def say_hello():
  print('Hello, world!')
    # | main.py |
# [ Importacion de un modulo ]
  import my_module      # Importacion del modulo 'my_module'
my_module.say_hello() # Utilizacion de la funcion 'say_hello' del modulo 'my_module'
  
# [ Importacion de conceptos de un modulo ]
from my_module import say_hello # importacion directa de 'say_hello' del modulo 'my_module'
say_hello()                     # Ya no es necesario anteponer 'my_module.'

    Utilizacion de modulos terceros

    Tambien podemos importar modulos de terceros dentro de un programa o script de Python si lo tenemos disponible. Osea, si lo tenemos instalado con pip o indicamos su ubicacion exacta

    # | main.py |
# [ Importacion de un modulo ]
import colorama           # Importacion del modulo 'colorama'
print(colorama.Fore.RED + # Utilizacion del dato 'Fore' del modulo 'colorama'
  'Hello, world!')

# [ Importacion de conceptos de un modulo ]
from colorama import Fore # importacion directa de 'Fore' del modulo 'colorama'
print(Fore.RED +          # Ya no es necesario anteponer 'colorama.'
  'Hello, world!')

    Para hacer mas sencillo entender como funciona la importacion de modulos. Python, al encontrar con un import nombre_del_modulo o from nombre_del_modulo import ..., primero revisa el directorio del programa o script que solicita la importacion del modulo para observar si encuentra dicho modulo en ese directorio. Si Python no lo encuentra en ese directorio, revisa en los directorios donde se almacenan los modulos instalados con pip o sus gestores de paquetes. Entre estos, generalmente se encuentran los modulos propios de la API de Python

    Nota: Podemos usar from nombre_del_modulo import * para importar directamente todos los conceptos del modulo (no muy recomendable), o from nombre_del_modulo import concept_1, concept_2 para importar directamente los conceptos indicados

    Nota: Generalmente, los modulos de terceros poseen la documentacion de uso del modulo en su propia entrada de PyPi o Github

    Paquetes

    Un paquete es una manera bastante interesante de almacenar varios modulos relacionados en un mismo directorio
    La estructura de un paquete es la siguiente

    .
├── Nombre_del_paquete/ # -> El nombre del paquete.
│   ├── __init__.py     # -> El archivo de inicio. Este archivo indica a
│   │                   # Python que se encuentra dentro de un paquete.
│   │                   # Tambien representa el primer script en ser
│   │                   # ejecutado al importar el paquete.
│   ├── saludos.py      # -> Un modulo del paquete.
│   └── despedida.py    # -> Otro modulo del paquete.
└── main.py             # -> Este archivo no pertenece al paquete.

    Nota: De la misma forma, un paquete puede contener subpaquetes dentro. Todos con la misma estructura en general

    Podemos importar este paquete desde un programa o script externo de la siguiente forma

    # | main.py |
from Nombre_del_paquete.saludos import saludar
# Asumiendo que en el modulo existe una funcion
# llamada 'saludar'.

saludar()

    Distribucion

    Aunque podemos implementar el uso de paquetes en nuestras aplicaciones, puede ser conveniente hacer del paquete un paquete distribuible
    Los paquetes distribuibles, son paquetes que pueden ser instalados en un sistema y almacenados en los directorios de paquetes y modulos. De esta manera, pueden ser importados en cualquier programa o script de Python sin tener que acceder manualmente desde ellos a los directorios de estos paquetes. Es de hecho, de esta manera en la que los paquetes son distribuidos en PyPi
    Para poder hacer un paquete distribuible, necesitamos, por supuesto, un paquete ya desarrollado e incluir fuera del directorio raiz de dicho paquete el archivo setup.py, el cual debe tener el siguiente contenido

    # | setup.py |
import setuptools # Importamos el modulo 'setuptools'

setuptools.setup(
  name="Nombre del paquete",             # -> Nombre del paquete
  version="0.1.0",                       # -> Version del paquete
  description="Descripcion del paquete", # -> Descripcion del paquete
  author="RukkaPlus",                    # -> Autor del paquete
  author_email="example@gmail.com",      # -> Email del autor del paquete
  packages=setuptools.find_packages(where="Nombre_del_paquete"),
  # packages=["Nombre_del_paquete", "Subpaquete_0", "Subpaquete_1"...]
)

    Nota: Para mas informacion referente al empaquetado y distribucion de paquetes, revise La guia oficial de empaquetado

    Finalmente, para generar el paquete, ejecutamos el siguiente comando en una terminal o linea de comandos

    > py setup.py sdist
    $ python3 setup.py sdist

    Esto generara un archivo comprimido (dependiendo del OS sera .zip o .tar.gz) distribuible

    Para instalar el paquete ejecutamos el siguiente comando

    > pip install paquete-0.1.0.zip

    Para desinstalar el paquete ejecutamos el siguiente comando

    > pip uninstall paquete-0.1.0.zip

    Para instalar el paquete ejecutamos el siguiente comando

    $ pip3 install paquete-0.1.0.tar.gz

    Para desinstalar el paquete ejecutamos el siguiente comando

    $ pip3 uninstall paquete-0.1.0.tar.gz

CSS

Cascading Style Sheets (CSS) es un lenguaje Web que permite manejar la estetica de una pagina Web. Este lenguaje toma los elementos de una pagina construida en HTML mediante selectores y permite cambiar sus propiedades estilisticas (como por ejemplo el color de texto, los margenes, los bordes del elemento, etc)

    Syntax

    Tipos de selectores

    Para manejar la estetica de un elemento es necesario seleccionarlo mediante un selector

    • Todos los elementos : *
    • Etiquetas : etiqueta
    • Atributos : [atributo] / [atributo="valor"]
    • Clases : .clase
    • ID : #id
    • Desendecia : selector_padre selector_desendiente
    • Pseudo-clase : selector:pseudo-clase
    • Pseudo-elemento : selector::pseudo-elemento

    Jerarquia de especificidad

    Se trata del nivel jerarquico que cada seleccion puede poseer. Cuanto mayor sea un nivel jerarquico, mayor sera la importancia de un cambio en la estetica

    1. !important
    2. estilos en linea
    3. clases / pseudo-clases / atributos
    4. elementos
    5. pseudo-elementos
    aside { /* Seleccionando a los elementos "aside" */
  ...

  ...
}
.nounique { /* Seleccionando a los elementos de clase "nounique" */
  ... !important /* Su cambio sera definitivo */
}

    Tipos de elementos

    Cada tipo de elemento tiene comportamientos que pueden variar entre si. Podemos definir el tipo de elemento mediante la propiedad display

    • block : Los bloques pueden ocupar el espacio de su contenido o lo que sus propiedades definan. Estos ocupan una linea entera (ejemplos: h1, h2, h3, h4, h5, h6, p, div, ul, ol, etc)
    • inline : Los elementos en linea ocupan el espacio de su contenido. Estos no se separan por lineas enteras (ejemplos: b, i, a, button, etc)
    • inline-block : Ocupan como un elemento en linea pero adquiere propiedades de los elementos bloque (por ejemplo, es posible modificarle el ancho y el alto)
    • flex : Son cajas flexibles. Tiene particularidades parecida a la de los bloques, pero son muy utiles como cajas flexibles
    • grid : Son cajas con gtillas. Tiene particularidades parecida a la de los bloques, pero permiten estructurar elementos en sus grillas
    • table : Define el comportamiento como una tabla
    • inline-table : Define el comportamiento como una tabla en linea
    • list-item : Define el comportamiento como un elemento de lista

    La propiedad outline

    Es una propiedad shorthand que define un borde que no afecta ni a la caja ni al DOM

    #element-0 { outline: solid blue 10px; }
#element-1 { outline: solid red 10px; }

    La propiedad position

    Es una propiedad que define la posicion de un elemento

    • static : Posicion por defecto de los elementos
    • relative : Posicion que ocupa espacio en el DOM (parecida a static) pero permite personalizar mas la posicion (agrega las propiedades: top, left, right, buttom y z-index). Tambien mantiene su espacio y punto de referencia original
    • absolute : Posicion que no ocupa espacio en el DOM ni respeta los espacios de otros elementos (por defecto se reposiciona por debajo del elemento anterior). Agrega las propiedades: top, left, right, buttom y z-index). Las propiedades de posicion (top, left, etc) se ajustan al viewport o al contenedor y no a su punto de referencia. Tambien, por defecto, ajusta su tamanio al contenido
    • fixed : Fija su posicion al viewport durante todo el scroll. Posee las mismas caracteristicas que absolute
    • sticky : Fija su posicion al viewport durante todo el scroll desde que tiene contacto con el borde. Posee las mismas caracteristicas que absolute
    #element-0 {
  position: static;
}
#element-1 {
  position: relative;
  left: 50px;
}
#element-2 {
  position: absolute;
  left: 150px;
}

    La propiedad overflow

    Permite controlar los desbordes de contenido. Algunas propiedades son:

    • visible : Propiedad por defecto. Permite que el texto sobresalga del contenedor
    • auto : Detecta si el texto sobresale del contenedor. Si es asi, agrega un scroll
    • scroll : Obliga a que el contenedor tenga un scroll
    • hidden : Oculta implicitamente el scroll
    #element {
  height: 200px;
  overflow: hidden scroll;
}

    Pseudo-clases

    Las Pseudo-clases son keywords agregadas a selectores que especifican un estado especial del elemento seleccionado

    • :hover : Cuando el mouse apunta a un elemento
    • :active : Cuando un elemento sea pulsado por el mouse
    • :link : Cuando un enlace no fue visitado
    • :visited : Cuando un enlace si fue visitado
    • :focus : En los input. Cuando se encuentra enfocado
    #element-0:hover { color: red; }
#element-1:active { color: red; } 
#element-2:link { color: red; }
#element-3:visited { color: red; }
#element-4:focus { color: red; }

    • ::first-letter : La primera letra de un texto. No funciona en elementos inline
    • ::first-line : La primera linea de un texto. No funciona en elementos inline
    • ::selection : La seleccion de un texto
    • ::after : Lo que va despues del elemento

    Las propiedades object-fit y object-position

    La propiedad object-fit nos permite trabajar con imagenes ajustandola a diferentes tamanios

    • contain : Ajusta su tamanio a las resoluciones reales de la imagen adaptandose al contenedor
    • cover : Ajusta su tamanio al contenedor recortando el espacio sobrante
    • none : Ajusta el tamanio a los valores por defecto. Le chupa un huevo todo
    • scale-down : Ajusta la imagen a la que se ajuste mejor al contenedor (si contain es mas pequenio, sera contain. En caso contrario, sera cover)
    • fill : Expande la imagen con tal de que ocupe todo el espacio del contenedor
    #element-0 > img { object-fit: contain; }
#element-1 > img { object-fit: cover; }
#element-2 > img { object-fit: none; }
#element-3 > img { object-fit: scale-down; }
#element-4 > img { object-fit: fill; }

    La propiedad object-position nos permite trabajar con imagenes junto a object-fit para definir su posicion principal

    #element-0 > img { object-fit: contain; object-position: right; }
#element-1 > img { object-fit: cover; object-position: bottom; }
#element-2 > img { object-fit: none; object-position: top; }
#element-3 > img { object-fit: scale-down; object-position: left; }
#element-4 > img { object-fit: fill; object-position: 30px; }

    Variables

    Son espacios en memoria que pueden variar mas adelante y pueden guardar un valor para utilizarlo en diferentes elementos
    Las variables se definen anteponiendo al nombre con --, y se usan con la funcion var()

    :root {
  --variable: red;
}
div { 
  background: var(--variable); /* <- red */
}

    La propiedad transform

    Esta propiedad nos permite realizar una transformacion a un elemento

    • translate(x, y) : Translada un objeto a otra posicion
    • scale(size_x, size_y) : Escala el tamanio de un objeto
    • skew(value_deg) : Deforma el objeto
    • rotate(value_deg) : Rota el objeto
    #element { transform: translate(25px); }

    Nota: Tambien existe una propiedad llamada clip-path, que pemite transformar un objeto para que tenga una forma deseada. Es posible buscar en el navegador clip-path generator

    La propiedad background

    Es una propiedad que define el fondo de un elemento

    • background-color : Define el color de fondo (rgb, rgba, hexadecimal, etc)
    • background-image : Define una imagen de fondo (url())
    • background-size : Define el tamanio del fondo (x, y, auto, cover, contain)
    • background-position : Define la posicion del fondo (left, top, bottom, right, center)
    • background-attachement : Define la posicion del fondo (scroll (toma como referencia al scroll. Es el valor por defecto), fixed (toma como referencia al viewport, asi que queda fijo), inherit)
    • background-clip : Define los bordes que tomara en cuenta el fondo (border-box, padding-box, content-box)
    • background-origin : Recorta la imagen de fondo para ajustarlo a su valor (border-box, padding-box, content-box)
    • background : Shorthand de todo lo anterior

    La propiedad filter

    Permite manejar los filtros de una imagen

    • none : Propiedad por defecto
    • blur(px) : Desenfoque de la imagen
    • brightness(0 - 1) : Brillo de la imagen
    • contrast(%) : Contraste de la imagen
    • drop-shadow(medidas) : La sombra de la imagen (no es lo mismo que el box-shadow)
    • grayscale(%) : La escala de grises
    • hue-rotate(deg) : Rotacion de colores
    • invert(%) : Invierte los colores
    • opacity(%) : La opacidad de la imagen
    • saturate(%) : Saturacion de la imagen
    • sepia(%) : Lo vuelve sepia
    • url("filter.svg") : Permite importar los efectos que hayamos preparado

    Flexbox

    Un Flexbox es una caja flexible. Un Flexbox esta compuesto de un Flex-container y de sus Flex-items, siendo el primero el contenedor del item flexible
    Los elementos de display: flex seran los elementos flexibles, mientras que el contenedor en si se comportara como un contenedor display: block
    Flexbox tiene dos ejes: el main-axis (eje x) y cross-axis (eje y)
    A su vez, el main-axis esta compuesto de un main-start (inicio) y main-end (final), esto apuntando de izquierda a derecha. Igualmente con el cross-axis, esta compuesto de un cross-start (inicio) y cross-end (final), esto apuntando de arriba a abajo

    #element-cont { display: flex }

    Cambiar direccion del main-axis

    La propiedad flex-direction nos permite cambiar la direccion del main-axis. Sin embargo, el cross-axis tambien sera cambiado de direccion (si el main-axis esta en vertical, el cross-axis estara en horizontal. Si el main-axis esta en horizontal, el cross-axis estara en vertical)

    • row : valor por defecto. Define el comportamiento a los items como filas
    • column : Define el comportamiento a los items como columnas
    • row-reverse : Igual que row pero en sentido contrario
    • column-reverse : Igual que column pero en sentido contrario
    #element-cont { flex-direction : row; }
    #element-cont { flex-direction : column; }

    La propiedad flex-wrap

    Nos permite modificar el comportamiento de los items cuando se ven afectados por la falta de espacio. Por ejemplo, cuando el viewport tenga menos espacio que el minimo necesario, que ubique los iems que no quepen hacia abajo o hacia arriba

    #element-cont { flex-wrap: wrap; }
    #element-cont { flex-wrap: wrap-reverse; }

    Existe una propiedad shorthand que define el flex-direction y el flex-wrap en una sola linea. Esta propiedad es el flex-flow

    Algunos ejemplos

    flex-flow: column wrap;
flex-flow: row-reverse wrap;
flex-flow: column-reverse wrap-reverse;

    La propiedad flex-grow

    Esta propiedad permite que un contenedor reparta el espacio sobrante del viewport entre todos sus items, o que un item especifico se lleve todo ese espacio sobrante

    .element-items { flex-grow: 1; }
    #element-item-specific { flex-grow: 1; }

    Las propiedades flex-basis y flex-shrink

    Cuando se trabaja con flex-items, ya no se usa la propiedad width, generalmente es utilizado flex-basis, puesto a que tiene incluso mayor importancia que la propiedad width, sin embargo, funciona para lo mismo
    La propiedad flex-shrink permite ajustar cuanto espacio un item debe sederle a los otros items si el espacio empieza a limitarse

    #element-item {
  flex-shrink: 2; /* Sediendo el doble de su espacio */
  }

    Si definimos un flex-shrink menor a 1, entonces va a ceder esa cantidad de espacio (que seria menor a lo que por defecto se trae). Y si la definimos como 0, no va a ceder nada de espacio cuando el viewport empiece a reducir su tamanio

    La propiedad order

    Con order es posible definir la posicion de un elemento. Cuanto mas alto sea, mas alejado estara del inicio (main-axis)

    #element-item-specific { order: 10; }

    Grid

    Es un tipo de elemento que facilita estructurar esteticamente a los elementos dentro de grillas

    Composicion de un Grid

    • Grid container : El contenedor Grid. Se comporta como un Block pero cambia el comportamiento de sus items
    • Grid item : Los items dentro del contenedor Grid
    • Grid cell : Las celdas del contenedor Grid
    • Grid tracks : Se trata de las columnas y filas
    • Grid area : Es un conjunto de celdas que puede contener un item
    • Grid line : Se trata de las lineas entre columnas y lineas

    Para que un Grid container tenga columnas y filas es necesario especificarlo con grid-template-rows y grid-template-columns

    #element-cont {
  display: grid;
  grid-template-rows: 75px 75px 75px;
  grid-template-columns: 75px 75px 75px;
}

    La propiedad gap

    La propiedad gap es un shorthand de gap-column y gap-row
    Se trata de una propiedad que separa a las celdas entre si dentro de un Grid container

    #element-cont {
  display: grid;
  grid-template-rows: 75px 75px 75px;
  grid-template-columns: 75px 75px 75px;
  gap: 30px;
}

    Las propiedades grid-column y grid-row

    Estas propiedades son asignadas a Grid items, y sirven para indicar el la linea donde inicia este item y la linea donde termina
    El shorthand grid-column abrevia grid-colum-start y grid-colum-end. Igualmente con grid-row que indica grid-row-start y grid-row-end

    #element-item-0 { grid-column: 1 / 3; }
#element-item-1 { grid-row: 2 / 4; }
#element-item-2 { grid-column: 2 / 4; }
#element-item-3 { grid-column: 2 / 4; }

    La funcion repeat()

    Es una funcion que repite un valor una cantidad determinada de veces. Su sintaxis es repeat(veces, valor)

    #element-cont {
  grid-template-rows: repeat(3, 75px); /* <- 75px 75px 75px */
  grid-template-columns: repeat(5, 100px); /* <- 100px 100px 100px 100px 100px */
}

    Grid dinamico

    Algunas propiedades y valores facilitan el dinamismo de un Grid

    • minmax(minimo, maximo) : Un valor que permite definir el tamanio minimo y el tamanio maximo. Ejemplo grid-template-rows: 20px 20px minmax(20px, 50px);
    • min-content : Un valor que indica el tamanio minimo del contenido o que el contenido necesita. Ejemplo minmax(min-content, 200px);
    • max-content : Un valor que indica el tamanio maximo del contenido. Ejemplo minmax(max-content, 200px);
    • auto-fill : Un valor que genera espacio si tiene espacio suficiente. Ejemplo grid-template-columns: repeat(auto-fill, 20px);
    • auto-fit : Un valor que escala espacio si es necesario. Ejemplo grid-template-columns: repeat(auto-fit, 20px);

    Las propiedades align-items y justify-items

    Transiciones

    Usando la propiedad transition podemos definir transiciones de cambios en elementos

    • transition-property : Puede ser all (todas las propiedades van a tener transicion. No es muy recomendado), none, property (Nombre de una propiedad que sufrira el cambio)
    • transition-duration : Indica la duracion de la transicion (puede ser en segundos o milisegundos)
    • transition-delay : Indica el tiempo que se taradara en empezar la transicion
    • transition-timing-function : Indica la curva de bezzier que se trazara en la animacion. linear, ease, ease-in, ease-out, ease-in-out, step-start, step-end, steps(int, start|end), initial, inherit
    • transition : Es el shorthand de todo lo anterior
    #element {
  transition-property: background;
  transition-duration: 1s;
}
#element:active {
  background: blue;
}

    Animaciones

    Las animaciones son parecidas a las transiciones, sin embargo deben ser programadas en un bloque @keyframes

    • animation-name : Aqui se indica el nombre la animacion ya programada
    • animation-duration : Indica la duracion de la animacion
    • animation-delay : Indica el tiempo que se taradara en empezar la transicion
    • animation-timing-function : Indica la curva de bezzier que se trazara en la animacion. linear, ease, ease-in, ease-out, ease-in-out, step-start, step-end, steps(int, start|end), initial, inherit
    • animation-iteration-count : Indica la cantidad de veces que debe repetirse (puede tener como valor infinite para que se repita infinitamente)
    • animation-fill-mode : Indica el resultado final. Puede ser backwards (finaliza quedando como empezo), forwards (finaliza con los valores y propiedades establecidos en el ultimo fotograma), both (comienza con los valores del primer fotograma de la animacion sin tomar en cuenta las propiedades establecidas por defecto)
    • animation-direction : Puede ser normal, reverse (empieza desde el final hasta el inicio de la animacion), alternate (empieza normal y luego regresa al inicio alternadamente) y alternate-reverse (lo mismo que el anterior pero empezando desde el final)
    #element {
  position: relative;
  background: red;
  animation-name: anim-example;
  animation-duration: 3s;
  animation-iteration-count: infinite;
}
@keyframes anim-example {
  from { left: 0; }
  to { left: 100%; background: blue; }
}

    Responsive design

    El Responsive design se trata de ajustar una pagina web para que se adapte a distintas resoluciones (por ejemplo, que la misma pagina web que viste desde un computador, se vea adaptado a un telefono con una resolucion menor)
    Esto se logra generalmente a partir de @media, que consulta al navegador informacion referente al computador cliente que lo ejecuta

    • all : Lo adapta a todos los dispositivos
    • print : Lo adapta a la hoja de impresion
    • screen : Lo adapta a una pantalla determinada (la resolucion, la orientacion, etc)
    • speech : Lo adapta a sintetizadores de voz

    Para el Responsive design se emplea @media screen y se usan los operadores logicos AND (and) y OR (or)
    Primero es necesario consultar si se estara trabajando con la pantalla @media screen y luego podemos emplear una condicion sobre esta

    @media screen and (max-width: 750px) {
  #element {
    /* Si la pantalla tiene una longitud maxima de 750px... */
    background: red;
  }
}
    @media screen and (max-width: 750px) and (min-width: 325px) {
  #element {
    /* Si la pantalla tiene una longitud maxima de 750px y un minimo de 325px... */
    background: red;
  }
}
